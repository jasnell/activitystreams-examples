Quick Overview

Activity Streams 1.0 focused on four types of artifacts: Object, MediaLink, Activity, and Collection.

Example 1: A Simple Activity, "Sally posted a note"

{
  "actor": {
    "id": "acct:sally@example.org"
  },
  "verb": "post",
  "object": {
    "objectType": "note",
    "id": "urn:example:notes:1",
    "displayName": "A Simple Note"
  }
}

Example 2: The Equivalent AS 2.0 Activity

{
  "@context": "http://asjsonld.mybluemix.net",
  "actor": "acct:sally@example.org",
  "verb": "post",
  "object": {
    "@type": "note",
    "@id": "urn:example:notes:1",
    "displayName": "A Simple Note"
  }
}

Example 3: MediaLink objects provided links to other resources:

{
  "objectType": "note",
  "icon": {
    "url": "http://example.org/image.png",
    "height": 100,
    "width": 100
  }
}

Example 4: All objects share a common set of base properties:

{
  "objectType": "note",
  "displayName": "A Note",
  "content": "<div>A simple note</div>",
  "published": "2014-12-12T12:34:56Z",
  "updated": "2014-12-12T12:34:56Z"
}

Example 5: The AS 1.0 Base Schema provided common verb and objectType definitions

{
  "objectType": "video",
  "displayName": "Cute little kittens",
  "embedCode": "<video width="320" height="240" controls="controls">...</video>",
  "stream": {
    "url": "http://example.org/my_video.mpg"
  }
}

What was missing

Implementation experience demonstrated that Activity Streams 1.0 was lacking in a number of areas: Internationalization, Accessibility, Extensibility, Verboseness, Modularity. Activity Streams 2.0 was designed to address these limitations.
