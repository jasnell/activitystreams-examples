Activity Vocabulary and JSON-LD

Activity Streams 2.0 is a Linked Data Vocabulary + a JSON-LD Serialization

* "objectType" becomes "@type", allows multiple values

* "id" becomes "@id"

* "@context" provides the JSON-LD @context definition

* Parsers are required to support Compact URIs (e.g. vcard:fn 
  is short for http://www.w3.org/2006/vcard/ns#fn)

* Publishers MAY use additional JSON-LD'isms, 
  Consumers MUST be able to understand

  For instance, the following is a valid Activity Streams 2.0 document:

[
  {
    "@type": [
      "http://activitystrea.ms/2.0/Activity"
    ],
    "http://activitystrea.ms/2.0/actor": [
      {
        "@id": "acct:sally@example.org",
        "http://activitystrea.ms/2.0/displayName": [
          {
            "@value": "Sally"
          }
        ]
      }
    ],
    "http://activitystrea.ms/2.0/object": [
      {
        "@id": "urn:example:posts:1"
      }
    ],
    "http://activitystrea.ms/2.0/verb": [
      {
        "@id": "http://activitystrea.ms/1.0/post"
      }
    ]
  }
]

* Publishers need to weigh the benefit of relying too much on JSON-LD 
  mechanisms. Need to keep consumers in mind. Keep it simple.

Things Added

The Activity Streams 2.0 Vocabulary includes stuff that used to be defined as extensions to Activity Streams 1.0 

* "inReplyTo" and "replies"

* Audience Targeting using "to", "bto", "cc", and "bcc"

* "priority"

* GeoLocation using "location"

* Collection paging using "next", "prev", "first", "last" and "current"

* "validFrom", "validAfter", "validBefore", "validUntil"

* "status", "instrument", "participant", "result", "scope"

* "language"

Things Changed

A few existing bits were updated. 

* "duration" now supports ISO 8601 durations (e.g. "duration": "PT1M")

* "tags" is renamed to "tag"

* "attachments" is renamed to "attachment"

* An object can now have more than 1 object type (using JSON-LD's "@type")

* "displayName", "title", "content" and "summary" are Natural Language Values 
  (meaning they can be expressed using more than one language in the document)

* "height" and "weight" are clarified to express device independent pixels

* MediaLinks are now Link objects (clarified definition)

Things Removed

A few bits were removed. 

* "upstreamDuplicates" and "downstreamDuplicates" are 
  deprecated (did anyone every actually use these?)
